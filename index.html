01

Équipe de 33

OllMark

Développeur et concepteur · Projet annuel Master 2

Sept. 2025 – Mars 2026

Studio de création de contenus marketing intégré à Ollca, une plateforme e-commerce existante. Le produit repose sur une approche orientée plugins dans Penpot : les fonctionnalités s'ajoutent à l'outil de design sans en modifier le cœur.

Mon périmètre

  • Des plugins Penpot développés en Angular, connectés aux données de la plateforme e-commerce : boutiques, produits, codes promo et QR codes.
  • Les microservices Spring Boot exposant ces données, consommés par les plugins via une API REST.
  • Un moteur de génération par templates piloté par des expressions JSONPath, configurable à l'exécution sans redéploiement.
  • L'intégration d'un assistant IA conversationnel avec Spring AI, capable de générer et de modifier des éléments graphiques, avec prévisualisation avant application.
  • L'export des designs vers Meta (Instagram, Facebook), avec adaptation des formats via les API des plateformes.
Java Spring Boot Spring AI Angular API REST Microservices Keycloak Penpot Plugins Scrum

02

Équipe de 15

AGORA

Développeur équipe plateforme · Projet annuel Master 1

Oct. 2023 – Juin 2024

Refonte complète d'une plateforme web de jeux de société en ligne, où les joueurs s'affrontent à distance en temps réel. Le développement était réparti entre un groupe dédié aux jeux et un groupe dédié à la plateforme, le socle sur lequel les jeux viennent se brancher.

Mon périmètre

  • L'interface principale de la plateforme : navigation, gestion des lobbys, affichage des jeux disponibles.
  • Un système de lobbys par jeu, pour lancer une partie entre amis ou rejoindre des joueurs inconnus.
  • La synchronisation temps réel via le protocole Mercure : les parties et l'interface se mettent à jour sans rechargement.
  • La phase de conception amont : rédaction des documents techniques (STB, DAL, cahier de test, analyse des risques).
PHP Symfony 6.4 Mercure MariaDB Tailwind CSS Docker Traefik Scrum Jira

03

En autonomie

Plateforme de jeux temps réel

Projet personnel

Sept. 2025 – Janv. 2026

Plateforme de jeux multijoueurs sans comptes utilisateurs : l'accès à une partie repose sur un lien de lobby partageable, ce qui supprime la friction à l'entrée. L'enjeu principal était la synchronisation de l'état de partie entre les joueurs en temps réel.

Mon périmètre

  • Un back end C# avec .NET 9 et ASP.NET Core, découpé en trois projets : un domaine qui définit les contrats, une couche d'infrastructure qui les implémente, et une API qui expose le tout.
  • La communication temps réel via SignalR : arrivée des joueurs dans un lobby, diffusion des actions, mise à jour de l'état de partie.
  • Un système de lobbys avec lien partageable, gestion des participants, lobbys privés protégés par mot de passe et lancement réservé au créateur.
  • Plusieurs jeux multijoueurs avec gestion des règles, des tours et de l'état de partie, une liste des lobbys mise à jour en direct, un historique des actions et un classement par jeu.
C# .NET 9 ASP.NET Core SignalR React Entity Framework Core SQLite API REST
